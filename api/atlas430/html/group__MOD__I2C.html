◆ i2c_transfer_start() [1/2]

void i2c_transfer_start ( const i2c_package_t *  pkg,
void(*)(i2c_status_t result)  callback 
)

Start an I2C transfer.

This function begins a new I2C transaction as described by the pkg struct. This function is nonblocking and returns immediately after the transfer is started. The status of the transfer can be polled using the i2c_transfer_status() function. Alternatively, a callback function can be executed when the transfer completes.

Note
Global interrupts must be enabled.
Parameters
pkgPointer to a package struct that describes the transfer operation.
callbackOptional pointer to a callback function to execute once the transfer completes or fails. A NULL pointer disables the callback.

Start an I2C transfer.

This function begins a new I2C transaction as described by the pkg struct. This function is nonblocking and returns immediately after the transfer is started. The status of the transfer can be polled using the i2c_transfer_status() function. Alternatively, a callback function can be executed when the transfer completes.

Note
Global interrupts must be enabled.
Parameters
base_addressMSP430-related register address of the USCI subsystem. can be USCI_B0_BASE - USCI_B1_BASE, EUSCI_B0_BASE - EUSCI_B3_BASE
pkgPointer to a package struct that describes the transfer operation.
callbackOptional pointer to a callback function to execute once the transfer completes or fails. A NULL pointer disables the callback.
